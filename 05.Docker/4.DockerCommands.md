# Docker Commands

## Docker Image Commands

List all images
> docker images
> docker image ls

Shows all images currently stored on your system.

Pull an image
> docker pull node:18-alpine

Downloads an image from Docker Hub (or another registry).

Build an image
> docker build -t my-node-app:latest .

Builds an image using the Dockerfile in the current directory.
The -t flag gives the image a name (and optional tag).

Remove an image
> docker rmi my-node-app --force
> docker image rm <id or name>

Deletes the image from your local system.

## Container Lifecycle Commands

List running containers
> docker ps
> docker container ls -a

List all containers (including stopped ones)
> docker ps -a

Run a container
> docker run hello-world

Creates and starts a new container from an image.

Run in background (detached mode)
> docker run -d nginx

The -d flag runs the container in the background.

Run interactively with a terminal
> docker run -it ubuntu /bin/bash

Opens a bash shell inside the container. (-i = interactive, -t = terminal)

Stop a container
> docker stop <container_id>


Start a stopped container
> docker start <container_id>

Remove a container
> docker rm <container_id>

View container logs
> docker logs <container_id>

when you run continer in detached mode then you can use this command to see that container logs

Execute a command inside a running container
> docker exec -it <container_id> bash

Gives you an interactive shell inside an already running container.

## Networking and Ports

Run with port mapping
> docker run -p 8080:3000 my-node-app

Maps host port 8080 to container port 3000.

List all Docker networks
> docker network ls

Create a new network
> docker network create mynet

## Volumes & Data Persistence

List all volumes
> docker volume ls

Mount a local directory into a container
> docker run -v $(pwd):/app my-node-app

Mounts your current directory (pwd) into /app inside the container.

## Cleanup Commands

Remove stopped containers
> docker container prune

Remove unused images
> docker image prune

Remove everything not actively used
> docker system prune -a

Frees up disk space by deleting unused containers, images, and volumes.

## Commonly Used Flags

-d → Detached (run in background)

-it → Interactive + TTY (for terminal access)

--name → Assign a name to the container

--rm → Automatically remove container when it stops

-p host:container → Map ports

-e VAR=value → Pass environment variable
